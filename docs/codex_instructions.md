\# Codex Instructions

You are helping build a \*\*cool original 2D JRPG-inspired game\*\* using \*\*TypeScript and Phaser 3\*\*.



This project is built through \*\*vibe coding\*\* with AI-assisted development inside \*\*VS Code\*\*.



Your job is to help design and implement:

\- game systems

\- architecture

\- gameplay mechanics

\- battle logic

\- UI

\- data structures

\- animation systems

\- content scaffolding



Always optimize for:



\- clarity

\- modularity

\- fast iteration

\- expandability

\- readable code

\- playable results early

\# Project Goals



Build a \*\*2D top-down JRPG\*\* featuring:



\- Character creation

\- Overworld exploration

\- NPC interaction

\- Dialogue system

\- Towns and dungeons

\- Turn-based battle system

\- Animated sprites

\- Inventory and equipment

\- Party members

\- Leveling and skills

\- Boss battles

\- Save/load

\- Stylish UI

---



\# Development Philosophy



This project is built through \*\*vibe coding\*\*, meaning:



\- Build systems quickly

\- Keep them modular

\- Improve them iteratively

\- Avoid overengineering early

\- Favor playable features over theoretical architecture

\- Allow placeholder assets

\- Allow refactors once patterns become clear



Always prefer \*\*a working system over a perfect design\*\*.



---



\# Coding Style Rules



\## General Rules



Write code that is:



\- readable

\- modular

\- extendable

\- easy for AI to continue working with



Avoid:



\- giant monolithic files

\- deep inheritance hierarchies

\- tightly coupled systems

\- hardcoded content



Prefer:



\- small focused modules

\- clear naming

\- predictable patterns

\- composition over inheritance



---



\# Architecture Rules



Always keep these separations:



Game Logic  

→ systems



Rendering  

→ scenes + sprites



Content  

→ data files



UI  

→ UI components



Combat Logic  

→ combat system



Never tightly couple \*\*battle calculations with UI rendering\*\*.
